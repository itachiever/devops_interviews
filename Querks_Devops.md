# DevOps 1st Round Interview Questions:
1. Tell me your introduction & day-to-day activities?
2. Which cloud are you using & which AWS services are you using?
3. Tell me what is the difference between private & public subnets.
4. What is a namespace in Kubernetes?
5. What is the role of the state file in Terraform?
6. What are the most useful Terraform commands?
7. Have you worked on Terraform? How do you manage the state file? What is the Terraform architecture you are using?
8. Same Terraform file: if two people are working on it, how do you know if someone has created resources or not?
9. Suppose you are working on a Terraform script for EC2 infrastructure creation. A team member makes changes in the AWS Console, but your Terraform template is different. How do you fetch those changes into your Terraform template?
10. What is your application technology stack?
11. Which CI/CD tool are you using?
12. Tell me step by step, from basic to end level, how you deploy your Java application on an EC2 server.
13. How do you connect to an EC2 instance?
14. In Terraform, how do you connect to ECR?
15. In Kubernetes, what is RBAC?
16. How do you monitor the logs for pods running in an EKS cluster?
17. Are you using volume mounts? Using volume mount services or without volume mounts?
18. What is PVC?
19. What is ADD in a Dockerfile? What is the difference between CMD and ENTRYPOINT?
20. Tell the command in Ubuntu: if a process is running, how do you kill that process?
Do you have any questions?
