## Senior DevOps Engineer (Public Cloud – Azure & GCP)

## Role summary

As a Senior DevOps Engineer (Public Cloud – Azure & GCP), you will lead the design, automation, and operations of secure and reliable cloud platforms that enable teams to deliver software at scale. You will set engineering standards for CI/CD and Infrastructure as Code, strengthen observability and incident response, and partner with development, security, and architecture teams to improve reliability, performance, and cost efficiency across Azure and Google Cloud.

## Key responsibilities

- Architect, build, and operate cloud platforms in Azure and GCP, including networking, identity/IAM, compute, storage, and governance controls.
- Lead the design and standardization of CI/CD pipelines (e.g., Azure DevOps, GitHub Actions, Jenkins) with policy-driven approvals, automated testing, artifact management, and safe deployment patterns (blue/green, canary, automated rollback).
- Own Infrastructure as Code (IaC) at scale (Terraform preferred), including reusable modules, environments, code review standards, drift detection, and automated provisioning.
- Design and operate Kubernetes platforms (e.g., AKS and GKE) including upgrades, scaling, ingress, secrets/configuration, multi-tenant patterns, and reliability hardening.
- Implement end-to-end observability (metrics, logs, traces) and reliability practices: define SLIs/SLOs, tune alerts, improve MTTR, and drive operational excellence.
- Lead incident response for high-severity events; coordinate triage, communication, and post-incident reviews, and drive corrective/preventive actions.
- Partner with Security to implement secure-by-default controls: secrets management, least privilege, vulnerability scanning, patching automation, and compliance evidence where required.
- Drive resiliency and continuity planning: backup/restore, disaster recovery strategies, capacity planning, and performance testing for critical services.
- Mentor engineers and influence technical direction through documentation, architecture reviews, and cross-team collaboration.

## Required qualifications

- 6+ years of experience in DevOps, SRE, Platform Engineering, Systems Engineering, or Software Engineering with significant cloud operations/automation ownership.
- Strong hands-on experience designing and operating production workloads in Azure and/or GCP (experience with both strongly preferred), including IAM, networking, compute, storage, and governance.
- Expertise with Infrastructure as Code using Terraform (or equivalent) including modular design, environment strategy, state management, and CI integration.
- Strong Kubernetes experience (AKS/GKE or upstream), including operational troubleshooting, upgrades, scaling, ingress, and workload reliability patterns.
- Advanced CI/CD experience: pipeline architecture, build/release standards, artifact/version management, and deployment safety practices.
- Strong Linux and networking fundamentals (TLS, load balancing, routing, DNS) and ability to debug distributed systems using logs/metrics/traces.
- Security-minded engineering: secrets management, RBAC/least privilege, vulnerability remediation, and secure automation practices.
- Proven ability to lead incidents and drive root-cause analysis through to measurable corrective actions.
- Excellent communication and stakeholder management skills; ability to mentor others and influence engineering standards.

## Preferred qualifications

- Experience implementing cloud landing zones / foundational platforms (Azure Landing Zone, GCP organization/folder projects) including guardrails, policy, and network topology.
- GitOps and platform delivery experience (Argo CD/Flux).
- Policy as code and security automation (OPA/Gatekeeper, Kyverno, Azure Policy, GCP Organization Policy, Terraform policy frameworks).
- Experience with service mesh, API gateways, and workload identity patterns (e.g., Workload Identity on GKE, managed identity on Azure).
- Experience operating multi-region or regulated environments, including audit/compliance evidence collection automation.
- Strong observability implementations using OpenTelemetry and modern logging/metrics stacks (Grafana/Prometheus, managed cloud monitoring).
