# Questions
1. Tell me about yourself.

2. What technologies are you currently working with and how many years of experience do you have?

3. How good are you in Azure?

4. If you have an RBAC role assigned but are still unable to access Azure Key Vault, why might that happen?

5. If your infrastructure uses private endpoints and you update a secret in Key Vault and want to deploy it through Azure DevOps without sending requests over the public internet, how would you deploy resources in this private endpoint scenario?

6. For CI and CD, would you recommend using a different repository for CD or keeping both CI and CD in the same repository?

7. Do you know how to create Docker images?

8. Can you write a basic Azure DevOps pipeline for the CD process with three stages: build artifact, test with SonarQube, and deploy using a self-hosted agent?

9. When multiple developers are working in a team, what is the best practice for tagging Docker images?

10. Can you explain the architecture of Kubernetes?

11. What is the function of Ingress in a Kubernetes cluster?

12. If you have RBAC configured in Azure and both of us are contributors in the same group, how would you restrict my access so that I cannot make changes to your Kubernetes cluster?

13. Can a Docker container start automatically?

14. What is the difference between a Docker image and a Dockerfile?

15. Do you have experience with Terraform?

16. Where do you store the Terraform state file in your setup?

17. In Terraform, have you broken down your `main.tf` into different modules, and how are you storing secrets?

18. In your Terraform modules and `main.tf`, which resources are you deploying?

19. Why are both AWS and Azure used in your setup?

20. Which resources are deployed in AWS and which ones are in Azure?

21. Does your Terraform infrastructure-as-code deploy to AWS or Azure?

22. How do you handle Terraform state lock issues when multiple teams and both AWS and Azure are involved?
