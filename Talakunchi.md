# Answers:

**1. Tell me about your job roles & responsibilities?**
-> I bridge Dev & Ops, automating CI/CD pipelines, Docker containers, and AWS infrastructure.

**2. Different components of DevOps?**
-> It's a culture of collaboration, automation, lean principles, measurement, and sharing (CALMS).

**3. Importance of feedback phase?**
-> It closes the loop, enabling rapid learning and continuous improvement for all teams.

**4. Difference between Waterfall, Agile & DevOps?**
-> Waterfall is linear, Agile is iterative for development, and DevOps extends Agile with automation for the entire lifecycle.

**5. What exactly Git & GitHub?**
-> Git is the local version control tool; GitHub is the cloud service for hosting Git projects.

**6. What is a pipeline?**
-> It's an automated assembly line that builds, tests, and deploys software from code to production.

**7. What is CI/CD?**
-> CI automates building/testing code on commit; CD automates releasing that tested code to users.

**8. What is a build tool? Why is it used?**
-> It automates compiling code, running tests, and packaging it, ensuring consistency and managing dependencies.

**9. What is Docker? Explain.**
-> A tool for creating containers, which are lightweight, isolated packages containing an app and all its dependencies.

**10. What is a Docker image?**
-> It's the static blueprint or template used to create a runnable Docker container.

**11. What is the difference between a Docker image and a container?**
-> An image is the static blueprint; a container is the live, running instance of that image.

**12. What is the 30% mentioned in your resume?**
-> I automated our manual deployment process, reducing the overall release cycle time by 30%.

**13. What is the POC mentioned in your resume?**
-> I led a Proof of Concept using Terraform to manage our AWS infrastructure as code, proving its effectiveness.

**14. What deployment failure occurred & how did you troubleshoot it?**
-> A deployment failed due to a missing config variable; I found it in the logs, reverted the change, and added a pipeline check.

**15. Entire troubleshooting process documented?**
-> Yes, I documented the entire process, solution, and preventative measures in our team's Confluence wiki.

**16. SAST & DAST (Dynamic & Static Application Security Testing)**
-> SAST scans source code for vulnerabilities (white-box); DAST attacks a running application to find vulnerabilities (black-box).

**17. What exactly is cloud technology?**
-> It delivers computing services like servers and storage over the internet on a pay-as-you-go basis.

**18. Services offered by AWS?**
-> Compute (EC2), Storage (S3), Databases (RDS), Networking (VPC), and Security (IAM).

**19. Detailed overview of AWS services?**
-> EC2 is a virtual server, S3 is infinite storage, VPC is a private network, and IAM controls user permissions.

**20. Benefits of EC2 instance?**
-> It offers scalability, flexibility, cost-effectiveness, and high reliability for running applications.

**21. What tool was used & what does Jenkins do?**
-> We used Jenkins, an automation server that orchestrates the entire CI/CD pipeline from code to deployment.

**22. Technical details on achieving effectiveness in deployment?**
-> Through full automation, containerization, Infrastructure as Code, and automated quality gates.

**23. Reason for job change?**
-> I'm seeking new challenges, an opportunity to work at a larger scale, and to deepen my expertise in advanced technologies.

**24. Do you have any questions, Feedback?**
-> Yes, I'd like to know about the team's current challenges, the collaboration culture, and the technology roadmap.
